# Use an existing embedding endpoint (Ollama or OpenAI-compatible)

This how-to shows you how to point Equibles at an embedding endpoint you already run yourself — for example, a shared Ollama on your GPU host, or an OpenAI-compatible service — instead of using the bundled `--profile embedding` Ollama container.

Use this when you want to avoid the extra ~3 GB Docker download, share a GPU across services, or use a managed embedding API. If you don't already run one, follow [Enable semantic search over SEC filings](how-to-enable-embedding-search.md) instead.

## Steps

1. **Make sure the embedding model is available on your endpoint.**

    For Ollama:

    ```bash
    # Replace with your Ollama host
    curl http://ollama.local:11434/api/tags
    ```

    The response should list a model. BGE-M3 is the default Equibles expects:

    ```bash
    ollama pull bge-m3
    ```

    For an OpenAI-compatible endpoint, confirm with the provider which embedding model name to use (e.g., `text-embedding-3-small`).

2. **Edit `.env`** and set the three embedding variables. Point `Embedding__BaseUrl` at your endpoint and use the matching model name:

    ```env
    Embedding__Enabled=true
    Embedding__BaseUrl=http://ollama.local:11434
    Embedding__ModelName=bge-m3
    # Optional — defaults to 10
    # Embedding__BatchSize=20
    ```

    If the host running Equibles can reach Ollama only through Docker's host network, use `http://host.docker.internal:11434` on macOS / Windows or your host's LAN IP on Linux.

3. **Start the stack without the bundled embedding profile.** The vanilla `docker compose up -d` is enough — `--profile embedding` would launch its own Ollama, which you don't want here:

    ```bash
    docker compose up -d
    ```

4. **Check progress on the Status page** at `http://localhost:8080/status`. The **chunks with embeddings** count should climb from zero. If it stays at zero, check `docker compose logs worker` for `Embedding endpoint returned`-style errors — the most common causes are an unreachable URL, the model not pulled on the endpoint, or a model-name mismatch.

5. **Try a semantic-search prompt** against your AI assistant once the first batch of chunks is embedded (a few minutes):

    *"Use the Equibles SearchDocuments tool to find passages in Apple's most recent 10-K about supply-chain risk. Summarise the top three matches."*

## Switching back to the bundled endpoint

Unset `Embedding__BaseUrl` (or comment it out) and bring the stack back up with the bundled profile:

```bash
docker compose --profile embedding up -d
```

Embeddings already generated by either endpoint stay searchable — only newly-generated vectors are affected.
